Title: Login

Primary Actor: User

Stakeholders and Interests:

User: Instructor, Tech, or Student that wants to log in securely

Instructor: wants to control access

Tech: wants log of access

Precondition: SuD presents login page.

Main Scenario:

  1. User types in login credentials
  2. SuD confirms login and records login attempt
  3. SuD advances to quiz selection

Extensions:

1a. User login window has a link that when clicked brings the user to a webpage where they can request a new password for their authentication account.

2.a The SuD confirms login and displays according access that the account is allowed

Variation: 1a. MUN credentials are supplied by browser
