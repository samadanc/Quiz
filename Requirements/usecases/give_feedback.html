Title: Give Feedback

Primary Actor: Instructor

Stakeholders and Interests:

Instructor: wants to be able to give feedback to students, especially for short answer questions where manual marking is required

Student: wants feedback from instructor regarding performance on overall quiz, or short answer questions if present

Tech: wants to enable the feature of written feedback for the instructor

Precondition: The deadline to submit the quiz has passed.

Main Scenario:

  1. Instructor logs in to verify credentials, and is granted administrator access
  2. SuD lists quiz submission for that particular class
  3. Instructor selects a quiz submission from a student
  4. SuD displays quiz submission and also displays "Give Feedback" button for each question
  5. Instructor clicks "Give Feedback" button
  6. SuD provides a text box for instructor to give feedback in, a "Submit" button to submit feedback, and a "Save" button to save feedback
  7. Instructor grants a written feedback of performance for that question and presses "Save" button
  8. Instructor clicks "Submit" button to submit feedback
  9. SuD records feedback and is displayed in Submissions for student to view
  10. Instructor repeats from step 4, or step 2 if a different student needs to be given feedback

Extensions:

1a. Instructor is not able to verify credentials

1a1. the page is refreshed and the Instructor is prompted again to verify credentials

8a. Instructor is not able to submit by clicking "Submit" button

8a1. Instructor is sent back to page of feedback where previous feedback has already been saved due to the "Save" button
