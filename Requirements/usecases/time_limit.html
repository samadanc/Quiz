Title: Time Limit.

Primary Actor: Instructor.

Stakeholders and Interests:

Instructor- wants a time limit to see if students can apply what they've learned efficently.

Student- wants to be able to manage their time, and to see what they need to work on.

Precondition: Teacher has created a quiz, but has not released it to students.

Trigger: Teacher wants to add a timer to their quiz.

Main Success Scenario:

  1. Instructor finds the quiz they have created in the SuD.
  2. Instructor selects option to add a timelimit.
  3. Instructor enters a timelimit in minutes to the quiz.
  4. Instructor applies changes.
  5. Instructor pushes quiz to students.
  6. Quiz is unlocked for the specified time period once students begin quiz.
  7. After the time limit has expired, any unsubmitted quiz will be automatically submitted.

Extensions:

2A: Time Limit has already been added to quiz.

2A.2: Option to change time limit offered.

7A: Student started but not finished all questions.

7A.1: Student's answers should be saved per use case Take Quiz, thus there marks will be calculated based on what's done at deadline.

7B: Student never began quiz, thus time limit never began.

7B.1: Refer to use case Take Quiz.
