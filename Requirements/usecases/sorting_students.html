Title: Sorting Students

Primary Actor: Instructor

Stakeholders and Interests:

Instructor: wants to sort student list by a quiz grade

Technician: wants operations to be traceable

Precondition: The deadline for submission has passed

Main Scenario:

  1. Instructor logs in the class quiz page
  2. SuD displays all the courses which the Instructor is teaching
  3. Instructor selects a specific course section
  4. SuD displays the list of students by last name
  5. Instructor selects the quiz which he/she wants to review the results
  6. Instructor clicks the "sort by grade (ascending)" button or the "sort by grade (descending)" button
  7. SuD sorts and displays the student list by quiz grade in ascending or descending order as instructor requires

Extensions:

1a. Instructor cannot log in the class quiz page

1a1. Instructor checks the user name and password

1a2. Instructor logs in the page again

5a. Instructor cannot find the quiz that he/she wants to review

5a1. Instructor checks the course section he/she selected and repeat step 3

7a. More than one student has the same grade

7a1. SuD sorts and displays these students by their last name
